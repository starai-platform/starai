# StarAI 渐进优化实施计划

> 原则：按 P0 → P1 → P2 顺序实施；每一阶段独立测试、构建、部署和回滚，不直接修改生产业务数据。

## P0：生产安全与资金一致性

- [x] 默认账号密码仅在数据库仍保留占位哈希时初始化，禁止部署迁移覆盖已修改密码。
- [x] 生产部署发现公开默认 JWT 密钥时显示醒目安全警告，但保留一键部署体验。
- [x] 生产环境禁止模拟支付，非生产环境仅接受 `mock` 渠道。
- [x] Worker 计费和解冻事务检查 Begin、Query、Exec、Commit 全部错误。
- [x] 计费按仍处于 `frozen` 的冻结记录执行，重复执行时安全跳过。
- [x] 任务和工作流只在计费成功后标记成功。
- [x] 余额、提现、卡密、模型、系统配置和冻结余额处置增加超级管理员限制。
- [x] 增加支付渠道和管理员角色中间件测试。
- [x] Go 测试、前后端 lint、Web/Admin 生产构建通过。

## P1：安全、部署与可观测性

### P1.1 会话安全

- [x] 停止普通用户 JWT 查询参数认证。
- [x] 增加 HttpOnly、Secure、SameSite Cookie 会话，并保留短期 Authorization 兼容期。
- [x] OAuth 回调改用 HttpOnly Cookie，避免 JWT 出现在浏览器地址和前端脚本中。
- [x] 管理端会话同步迁移，保留旧 Authorization 会话兼容。
- [x] 增加 Cookie 登录、登出失效和敏感操作权限集成测试脚本。
- [ ] 在预发布环境继续执行 OAuth 提供商真实回调和浏览器跨站回归测试。

### P1.2 限流与滥用防护

- [x] 增加 Redis 原子限流器，Redis 故障时自动放行。
- [x] 覆盖登录、注册、验证码、管理员登录、普通用户 API 和 OpenAI API。
- [x] 分别按 IP、用户、API Token 设置阈值，并返回标准 `429` 与重试时间。
- [x] 配置可信反向代理，防止伪造 `X-Forwarded-For` 绕过限流。

### P1.3 迁移与无停机部署

- [x] 每个迁移文件在独立事务中执行；失败时不得把整个文件标记为完成。
- [x] 为已执行迁移记录 SHA-256 校验和，禁止修改已上线迁移文件。
- [x] 增加迁移文件编号重复、校验和变化和缺少 down 文件的 CI 检查。
- [x] API、Web、Admin、Worker 增加健康检查。
- [x] 默认构建期间不停止当前容器，并在重建后等待健康检查。
- [x] 保存上一版本应用镜像，健康检查失败时自动恢复旧镜像；数据库迁移不自动逆向回滚。

### P1.4 可观测性

- [x] 增加全链路 request ID 和结构化 HTTP 访问日志。
- [x] 增加 HTTP 请求量、5xx、耗时、并发和限流指标。
- [x] Worker 心跳进入 API 健康状态和 Prometheus 指标。
- [x] 提供 Worker 心跳、HTTP 错误率和限流突增告警规则示例。
- [x] 增加支付回调拒绝和内容安全拦截业务指标及告警示例。
- [ ] 继续扩展上游模型、队列积压、计费失败和存储异常业务指标。

## P2：功能闭环与体验

- [x] 增加项目删除、归档、复制和自定义风格删除。
- [x] 支持失败节点单独重试，复用成功阶段输出，避免整条工作流从头生成。
- [x] 删除执行中项目时进行保护，重试入队失败时释放冻结余额。
- [x] 保存部分成功的关键帧和视频分段，并支持按索引人工替换后重新合成。

- [ ] 端到端验证完成后移除“功能开发中”标记。

### P2.3 测试与质量门禁

- [x] 增加卡密并发兑换、提现并发、会话和漫画生命周期数据库集成测试脚本。
- [ ] 继续增加迁移校验和变化、任务取消/重试数据库集成测试。

- [x] CI 执行 Go test/vet、TypeScript、ESLint、生产构建和迁移配对校验。

### P2.4 支付、内容安全与灾备

- [x] 增加默认关闭的关键词拦截，并仅记录命中词哈希。
- [x] 完整备份增加 SHA-256 清单、危险路径检查和只读验证脚本。
- [ ] 接入选定支付商的官方 SDK、退款、关单和每日对账。
- [ ] 在隔离主机执行完整恢复演练，并记录恢复点目标与恢复时间。

## 阶段验收规则

1. 不在测试中使用真实生产用户、余额、订单或模型密钥。
2. 所有数据库变更必须有向前迁移、回滚说明和备份要求。
3. 每阶段必须通过 Go 测试、前后端静态检查和生产构建。
4. 涉及登录、支付、计费和部署的改动必须先在预发布环境验证。
5. 当前阶段未验收前，不进入下一阶段生产部署。
