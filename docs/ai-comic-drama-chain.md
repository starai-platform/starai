# AI 漫剧链路与安全上线说明

## 当前主链路

```text
项目/风格/角色/道具/场景
  → 剧情与分镜规划
  → 分镜资产 code 绑定
  → 关键帧生成
  → 分段视频生成
  → FFmpeg 合成与独立封面
  → 作品入库与结算
```

角色、道具、场景和分镜使用 `049_comic_drama_assets` 增量迁移持久化。旧项目仍从
`workflow_projects.outputs` 读取，不需要转换旧数据。

## 运行保护

- 漫剧父任务超时为 6 小时，数据库咨询锁阻止同一项目被并发消费。
- 逐步确认任务可在前台继续、切换自动托管或取消；取消会释放冻结额度。
- 页面刷新或重新选择项目后，会从最近工作流恢复状态与结果。
- 重试保留历史节点及已发生费用，避免复用结果时漏计成本。
- 替换关键帧会使下游视频和成片失效；替换片段会使成片失效。
- 媒体替换只接受当前用户上传后的 `asset_id`，不下载任意外部 URL。

## 资产一致性

- 管理的角色、道具和场景具有稳定 code、版本、状态和视觉提示词。
- 分镜通过 `character_codes`、`prop_codes`、`location_code` 引用资产。
- 关联资产提示词、项目风格提示词、风格封面和用户多参考图会传给生成模型。
- 没有配置真实视觉质检模型时，系统返回 `checked=false`，不会伪造一致性分数。

## 上线顺序

1. 先运行迁移 `049_comic_drama_assets.up.sql`。
2. 部署 API 与 Worker，再部署 Web；旧 Worker 不依赖新表，新 Worker 对沉淀失败采用兼容降级。
3. 保持 AI 漫剧功能开关现状，在预发布账号验证自动托管和逐步确认各一次。
4. 核对钱包冻结/释放、关键帧数量、片段数量、封面 URL 和最终成片。

## 仍需管理员明确配置后才能开启的能力

- 真实视觉质检与低分镜头自动重试：需要单独的多模态质检模型，避免隐性消耗额度。
- TTS、角色声线、字幕、BGM、音效和混音：需要明确音频模型及计费规则。
- 并行镜头生成：需要按上游并发限制配置，默认仍采用稳妥的顺序生成。

以上能力不配置时采用明确降级，不阻塞现有漫剧生成和正常运营。
